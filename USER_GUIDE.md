# ContestTrace 用户指南

## 1. 系统简介

ContestTrace是一个学科竞赛信息聚合平台，旨在帮助用户方便地获取和管理高校学科竞赛信息。系统通过爬虫技术自动收集各学院和部门的竞赛通知，经过智能筛选和处理后，为用户提供结构化、易访问的竞赛信息。

## 2. 安装与配置

### 2.1 环境要求

- Python 3.7 或更高版本
- pip 包管理工具
- 互联网连接（用于爬虫运行）

### 2.2 安装步骤

1. **克隆仓库**
   ```bash
   git clone <repository-url>
   cd ContestTrace
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **配置文件**
   系统使用以下配置文件：
   - `config/sites.yaml`：配置爬虫目标网站
   - `config/config.yaml`：主配置文件
   - `config/model_config.yaml`：模型配置文件

   这些文件已经包含默认配置，一般情况下无需修改。

## 3. 运行系统

### 3.1 一键运行

运行 `run.bat` 文件，会显示以下菜单：

```
=======================================
       ContestTrace Menu
=======================================
1. Install Dependencies
2. Run Spiders
3. Merge Raw Databases
4. Filter Data
5. Export Data
6. Start Frontend Server
7. Run Full Process (Spiders->Merge->Filter->Frontend)
0. Exit
=======================================
Please enter option number:
```

### 3.2 操作说明

1. **Install Dependencies**
   - 安装项目所需的Python依赖包
   - 首次运行时建议执行此操作

2. **Run Spiders**
   - 运行爬虫，爬取各学院和部门的通知公告
   - 会生成多个原始数据库文件，如 `data/contest_trace_raw_新闻与传播学院.db`

3. **Merge Raw Databases**
   - 将多个原始数据库汇总为一个主原始数据库 `data/contest_trace_raw.db`
   - 为后续的筛选操作做准备

4. **Filter Data**
   - 从主原始数据库中筛选出竞赛相关的通知
   - 生成竞赛数据库 `data/competiton.db`
   - 同时导出数据到前端 `contesttrace/frontend/data/contests.json`

5. **Export Data**
   - 将竞赛数据导出为CSV格式，存储在 `data/exports/` 目录
   - 方便进行进一步的数据分析

6. **Start Frontend Server**
   - 启动前端服务器，默认端口为8000
   - 在浏览器中访问 `http://localhost:8000` 查看竞赛信息

7. **Run Full Process**
   - 执行完整流程：爬虫->汇总->筛选->前端
   - 一键完成所有操作，适合初次运行或定期更新

## 4. 前端使用指南

### 4.1 访问前端

在浏览器中访问 `http://localhost:8000`，即可看到ContestTrace的前端界面。

### 4.2 功能说明

#### 4.2.1 竞赛列表

- **搜索功能**：在搜索框中输入关键词，可搜索竞赛标题和内容
- **筛选功能**：
  - **来源筛选**：选择特定学院或部门的竞赛
  - **时间筛选**：选择最近一周、一个月或三个月的竞赛
  - **月份筛选**：选择特定月份的竞赛
- **分页功能**：浏览多页竞赛信息

#### 4.2.2 竞赛详情

点击任意竞赛卡片，会弹出竞赛详情模态框，显示以下信息：
- 竞赛标题
- 来源
- 发布时间
- 截止时间
- 组织者
- 参赛对象
- 奖项设置
- 联系方式
- 详细内容
- 竞赛链接

#### 4.2.3 数据统计

在统计页面，可查看：
- 总竞赛数
- 本月新增竞赛数
- 竞赛月份分布图表

#### 4.2.4 个人设置

在设置页面，可：
- 开启/关闭通知提醒
- 设置感兴趣的关键词
- 导出竞赛数据为CSV格式

## 6. 常见问题

### 6.1 爬虫运行失败

**问题**：爬虫运行时出现错误

**解决方法**：
- 检查网络连接是否正常
- 检查目标网站是否可访问
- 查看日志文件，了解具体错误信息

### 6.2 筛选结果不准确

**问题**：筛选出的竞赛信息不准确或遗漏

**解决方法**：
- 检查 `competition_filter.py` 中的关键词配置
- 调整筛选规则，提高筛选准确性

### 6.3 前端无法加载数据

**问题**：前端页面无法显示竞赛数据

**解决方法**：
- 检查 `contesttrace/frontend/data/contests.json` 文件是否存在
- 检查文件格式是否正确
- 重新运行筛选操作，导出最新数据

### 6.4 数据库连接失败

**问题**：系统无法连接数据库

**解决方法**：
- 检查数据库文件是否存在
- 检查数据库文件权限是否正确
- 尝试重建数据库

## 7. 高级功能

### 7.1 自定义爬虫

如果需要添加新的爬虫目标，可以：
1. 在 `contesttrace/core/spiders/` 目录下创建新的爬虫文件
2. 继承 `BaseSpider` 类，实现必要的方法
3. 在 `SpiderManager` 中注册新爬虫

### 7.2 自定义筛选规则

如果需要调整筛选规则，可以修改 `contesttrace/core/filter/competition_filter.py` 文件：
- 添加或修改强竞赛词
- 添加或修改排除关键词
- 调整置信度计算逻辑

### 7.3 自定义前端

如果需要修改前端界面，可以编辑 `contesttrace/frontend/` 目录下的文件：
- `index.html`：修改页面结构
- `css/style.css`：修改样式
- `js/main.js`：修改交互逻辑
- `js/data.js`：修改数据处理逻辑

## 8. 故障排除

### 8.1 查看日志

系统运行过程中的日志信息会输出到控制台，同时也会记录到 `logs/` 目录下的日志文件中。如果遇到问题，可以查看日志文件了解具体错误信息。

### 8.2 重置系统

如果系统出现严重问题，可以尝试：
1. 删除 `data/` 目录下的所有数据库文件
2. 重新运行完整流程
