# ContestTrace 技术方案与国内外研究现状（简述）

## 1. 国内外研究现状（概述）

高校竞赛信息管理长期存在「通知分散、口径不一、检索困难」等问题。国内外高校信息化研究普遍认同：**数据采集规范化 + 轻量结构化存储 + 可视化呈现** 可显著提升信息触达效率。Web 信息抽取领域常用 HTML DOM 解析与规则/机器学习结合的混合方案，以兼顾可解释性与鲁棒性。

## 2. 本系统技术方案

1. **多源配置化爬虫**：以 YAML 描述列表页/详情页 CSS 选择器，降低站点变更带来的维护成本；通过 `robots.txt` 与随机延时满足合规要求。  
2. **编码自动识别**：采用 `charset-normalizer` 检测 + 多编码回退链，避免中文乱码（参见 `encoding_fixer.py`）。  
3. **SQLite 持久化**：以详情 URL 为唯一键去重；扩展字段覆盖赛事级别、类别、附件等结构化信息。  
4. **数据处理**：`difflib.SequenceMatcher` 用于正文相似度分析（Python 标准库文档）；`scikit-learn` 朴素贝叶斯 + TF-IDF 作为可复现的文本分类基线。  
5. **推荐**：默认规则引擎；可选 OpenAI 兼容接口实现语义推荐（API Key 仅来自环境变量）。  
6. **前端**：Streamlit 轻量化界面，降低部署门槛。

## 3. 实验与复现建议

- 运行爬虫入库后，在 Web 端执行「批量清洗」「jieba 标签」观察字段变化。  
- 使用 `data/training_data/sample_labeled.csv` 扩充样本后调用 `ml_model.train_and_save` 训练模型（需自行编写脚本或在交互环境中调用）。
